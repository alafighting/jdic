How to Contribute

  • Participate on Forums and Mailing Lists
    Through the JDIC forums and mailing lists, people using the software can benefit from each others' expertise. To be really useful, the lists need your input! Please don't be afraid to post — help out others when you can; post your questions, comments, and ideas as you see fit.
  • File a Bug Report or Feature Request
    Bug reports are very helpful to developers and a great way to contribute to a project! Remember to take a little time when filing a bug report — make sure you can reproduce the problem, and be very explicit in your description of the problem. Enter "Defect" or "Enhancement" for a bug report.

    If there's something you'd like to see and don't, or something you can imagine being improved, let us know by filing a feature request. Use the project's Issue Tracking and file a "Feature" rather than a "Defect" or "Enhancement". Besides a general summary of the proposed feature, please state clearly why this feature is required or desirable, and user cases this feature fits into.
  • Vote for a Reported Bug or Feature Request
    If your bug or feature request has already been reported, you can vote on it to prioritize it. Click on the Vote for this issue link directly under the Additional Comments field and enter the number of votes you want to cast in the votes field. After you've entered your desired number of votes, click on the Submit button.
  • Contributing Code
    As mentioned above, JDIC is open source — all of the source code is available, and if there's a feature you'd like to see or a bug you'd like to see fixed, you have the power to make that happen. See the sections on contributing patches for more information.

List of Requested Features

List of Requested Features - Besides the features available and those we are working on, lots of new feature requests coming from the project Forum, Issue Tracker, and feedback. All the requested features are tracked via issues in Issue Tracker.

New requested features should be reported as issues in Issue Tracker with the Issue type set as FEATURE. You can vote on an already reported feature. Comments or patch to these features are appreciated! Work on them could be held as incubator projects in the project Incubator Projects section. 

To hold an incubator project, please post to the project Forum, and drop an email to owner@jdic.dev.java.net (alias for the project owners). 

Build A JDIC Distribution

Build A JDIC Distribution - Directions for creating your own milestone build or any other distribution. You might want to create your own distribution in order to create a build for some platform that's not normally on our ftp site or to enable some feature that's not supported in the standard builds.

We like to provide up to date build documentation. If you find something to be out of date, incorrect, or missing altogether, send mail to georgez@dev.java.net of the build document in question or with the relevant current, correct, or completed information.

JDIC Releases - Release notes for all the major JDIC releases (major.minor.micro) and some sub-releases (major.minor.micro.*). It also contains the information for scheduled releases on the roadmap.
