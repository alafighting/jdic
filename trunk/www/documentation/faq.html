1. General

Q. What is JDIC?
A. JDIC is short for JDesktop Integration Components. The project aims to make Java technology-based applications (Java applications) first-class citizens of current desktop platforms without sacrificing platform independence. The JDIC project home page is: https://jdic.dev.java.net.

Q. When was JDIC made public?
 A. The JDIC project was first open sourced during JavaOne 2004 (June 28 - July 1). The code is available via CVS, and periodic binary releases. 

Q. What is the current version of JDIC?
A. JDIC source is continually available via CVS, and the latest versioned binary release is available here. 

Q. Under which license terms was JDIC released?
A. JDIC is released under LGPL. Take a look at JDIC's COPYING file for detailed license terms.

Q. What are the supported platforms?
A. Briefly, Windows, Linux and Solaris. Refer to the JDIC System Requirements for specific versions and distributions. Some contributors provided patches and binaries for the support on Mac OS X, FreeBSD and other platforms. Refer to the Release Notes of 0.9 and Mac OS X and FreeBSD support discussion threads for more information. There are still gaps for Mac OS X support waiting to be closed. We'd very much welcome any volunteers or effort on that.

The incubator projects may have different platform support matrixes, refer to the JDIC Incubator Projects space for their specific system requirements.

Q. How about Mac?
A. Refer to the "What are the supported platforms?" question. 

Q. Isn't Swing enough? Why are you introducing yet another desktop toolkit?
A. JDIC is not a replacement for Swing/AWT (that is, a GUI widget library). It's a set of APIs and tools to integrate with system services. It's compatible with Swing: you can use the components under Swing or AWT. It helps to make Java a more complete platform by providing new APIs that gives portable access to some useful system services.

Q. What are the incubator projects?
A. Some projects are associated with JDIC but aren't part of the source or binary distribution of the JDIC project. These incubator projects are released as separate downloads and develop at their own speed with different quality criteria. Once an incubator project complies with the requirements of JDIC, it might move into the JDIC project.

Q. Can I add an incubator project? How?
A. Yes. We consider the services currently available in JDIC a starting point rather than complete. We'll welcome efforts to address other desktop integration aspects. Check out the Incubator Projects section for all the existing incubator projects and detailed instructions to setup an incubator project.

Q. Is there any Java package naming convention?
A. Yes. All the Java packages of the JDIC project, including the incubator projects, use the top package name prefix org.jdesktop.jdic. The first package name segment after org.jdesktop.jdic is generally the component or incubator project name. The following package names are used now:

  • org.jdesktop.jdic.desktop - Desktop Component: Launches desktop programs to open, edit, print, mail and browser files.
  • org.jdesktop.jdic.filetypes - Filetypes Component: Sets desktop file type associations.
  • org.jdesktop.jdic.browser - Browser Component: Embeds a native web browser (Internet Explorer or Mozilla) into an AWT canvas.
  • org.jdesktop.jdic.tray - Tray Icon API: Support for system tray icons and pop-up menus.
  • org.jdesktop.jdic.packager - JDIC Packager: Command-line tools for converting Java Web Start programs into native installers.
  • org.jdesktop.jdic.screensaver - SaverBeans SDK incubator project
  • org.jdesktop.jdic.dock - Floating Dock Toplevel Window incubator project
  • org.jdesktop.jdic.icon - IconService incubator project
  • org.jdesktop.jdic.systeminfo - SystemInfo incubator project
  • org.jdesktop.jdic.misc - Misc incubator project
  • org.jdesktop.jdic.fileutil - FileUtil incubator project
  • org.jdesktop.jdic.mpcontrol - Music Player Control API incubator project

2. Installation and Configuration

Q. Where can I get a binary?
A. JDIC can be fetched as a binary in a versioned release or as a biweekly binary build. You could find the JDIC binary downloads, including the incubator projects downloads here. 

Q. Where can I get the source?
A. There are several ways you can access the JDIC source code:

  • Browse
    Browse the up-to-the-minute latest version of the source code online.
  • Download
    • FTP - Download the source zip file for a specific release.
    • CVS - The latest source could be checked out using CVS following the CVS client setup instructions. It lets you get up-to-the-minute changes. For each JDIC release, a tag is created to help people pull the right source. The tag names are of the form  "JDIC_major_minor_micro". For example, JDIC release 0.8.4 has a tag "JDIC_0_8_4".

Q. Why is there a defmailer.properties file in the binary distributions for Linux/Solaris?
A. This property file is used to define the default mail program path. While the user uses Desktop.mail() method, it tries to find the system default mail program in the GConf database. If the default mail program is not set, the method looks in this property file for the default mail program path.

Q. What are the embeddable browsers by package org.jdesktop.jdic.browser? how about FireFox?
A. On Windows, IE or Mozilla is embeddable. If Mozilla is set as the default browser, it will be embedded. Or else, IE will always be the embedded browser. On Linux/Solaris, Mozilla is embedded if it's set as the default browser or its executable mozilla could be found under PATH. The package has been tested with IE 5.0 and 6.0, Mozilla 1.4, 1.6, 1.7 and 1.8a4. Other browsers including FireFox, Galeon, Epiphany, Opera, etc. are not supported or tested, though they might work. There is a bug(#262453) on embedding Firefox. Once this bug is fixed, this package will support it.

3. Building the Source

Q. Can I pull a copy of the source and build my own binary?
A. Yes. After you pull the source code, follow the Build the Source instructions to build your own binary. We like to provide up to date build documentation. If you find something to be out of date, incorrect, or missing altogether, report a bug using JDIC Issue tracker, or send a mail to jdic@jdic.dev.java.net.

Q. What type of build system does JDIC use?
A. JDIC uses Apache Ant as the build system on all platforms.

Q. Can I build the JDIC source without building all the Mozilla stuff?
A. For Windows, the answer is no. The JDIC org.jdesktop.jdic.browser package makes use of some Mozilla code, some native code won't compile unless the Mozilla stuff it depends on has been built. For Linux/Solaris, after release 0.8.6, the answer is yes, the package builds against Gecko SDK and a Mozilla binary, both of which are downloadable from mozilla.org. Refer to Build the Source instructions for details.

Q. Why the Microsoft Platform Software Development Kit (SDK) Update is needed to build the source on Windows platforms?
A. The SDK Update is needed for below reasons:

  • The JDIC API native JNI code on Windows uses a Windows Shell function AssocQueryString to search for and retrieve a file association-related string from the registry. This functions is supported in the Platform SDK Update.
  • The JDIC Packager on Windows uses the latest Windows MSI SDK provided by the SDK Update.
  • The JDIC Packager on Windows uses a sample MSI template contained in the SDK Update. The sample template is used as a raw MSI template to be modified to generate the final MSI file.
  • The JDIC Packager tool jnlp2msi uses two tools provided by the SDK Update: msidb.exe and makecab.exe.

5.Troubleshooting Using JDIC

Q. Why there are problems using JDIC Browser in a Swing application?
A. JDIC Browser was implemented as an AWT (Canvas) component. Mix using it with Swing components have some known heavy/lightweight interoperability issues, which are stated in details in this article.

Q. The Browser demo doesn't work, why?
A. Follow below steps to check it:

  • Strictly follow the installation instructions.
  • JDIC Browser component only supports/embeds the default browser IE/Mozilla on Windows and Mozilla on Linux/Unix. Refer to details on the supported browsers.
  • On Windows, if Mozilla is the default browser, remove MozEmbed.exe file if it exists in the mozilla.exe directory (if Mozilla is unzipped from a .zip package) or the created GRE directory [Program Files\Common Files\mozilla.org\GRE] (if Mozilla is installed from an .exe package), which might be be installed by JDIC release 0.8.6 or earlier.
  • MOZILLA_FIVE_HOME environment variable is not used by 0.8.7 or later, please just unset it. If it's set (required by release 0.8.5 or earlier), make sure it points to a valid Mozilla binary path.
  • Help debug the problem and report the result:
    • On Windows, set IE as the default browser first and then Mozilla.
    • Uncomment below statement 
        ......
        webBrowser.setDebug(true); 
        ......
      in the Browser.java or SimpleBrowser.java demo source file to enable debug messages. If there is error message "Can't execute native browser. Error message is xxx", the native browser executables (IeEmbed.exe and MozEmbed.exe for Windows releases, and mozembed-<os>-gtk<version> for Linux/Unix releases) is not put under PATH, which is done by the code by default if they locate at the same place as jdic.jar  If there is no error message, it's quite possible that the default browser is not a supported embeddable browser (IE/Mozilla on Windows and Mozilla on Linux/Unix).
    • On Linux/Unix, the gtk version (1.2 or 2.0) of mozilla you run the demo with.
      This can be a problem if you build the JDIC source and run it with a mozilla binary youself. The gtk version of mozilla you run the demo with must be the same with the gtk version of mozilla you used to build the source. Check the gtk verion of mozilla: launch mozilla -> input "about:buildconfig" in the top address field -> check "Configure arguments", if there is an option "--enable-toolkit-gtk2", the gtk version is 2.0, or else, it's 1.2.

Q. Why the tray icon demo runs without any error on my Linux/Solaris platform, but there is no tray icon created?
A. The Notification Area is probably disabled. You can enable it by right-clicking the GNOME panel and selecting Add to Panel -> Utility -> Notification Area.

Q. java.lang.UnsatisfiedLinkError: no jdic in java.library.path
A. It's caused by that JNI can't find the native lib of JDIC,ways to fix this issue:
- For apps run in IDE, use -Djava.library.pah parameter to specify the location of libjdic.so/jdic.dll
- For win, put the folder contains jdic.dll into the path
- For solaris/unix/linux, put folder contains libjdic.so into the LD_LIBRARY_PATH

6.Troubleshooting Building the JDIC Source

Q. Why there is an error message "Makefile:42: /config/autoconf.mk: No such file or directory." on Unix, and "[exec] ...\msgserver.cpp(23) : fatal error C1083: Cannot open include file: 'prlog.h': No such file or directory" on Windows?
A. The environment variable MOZILLA_SRC_HOME is not set to the Mozilla source directory including a built binary. Refer to the JDIC Build the Source instructions.

Q. Why there is an error message "[exec] WinAPIWrapper.obj : error LNK2001: unresolved external symbol __imp__AssocQueryStringA@24" on Windows?
A. Microsoft Platform Software Development Kit (SDK) Update is not installed or set correctly. Download the SDK update from here, install it, and add <Microsoft SDK Update directory>\include and <Microsoft SDK Update directory>\lib before other include or library directories. By default, <Microsoft SDK Update directory> is C:\Program Files\Microsoft SDK, and the include/library directory is automatically set.

Q. Why there is an error message "GnomeBrowserService.cpp:21:17: jni.h: No such file or directory" on Unix?
A. The environment variable JAVA_HOME is not set to the J2SDK home directory.

8. Technical Questions and Answers for Browser

Q. How to choose IE or Mozilla browser?
A. You have two ways to choose a web browser, set by program or changing registry value for win.
1.Change win registry value.Open win "regedit", go to "HKEY_CLASSES_ROOT\HTTP\shell\open\command" and set "default" to the full path of you wanted browser.
2.Set by program. Before loading a page that uses WebBrowser instance, set the active brower engine through BrowserEngineManager.Sample as below:
public class ChooseBrowserSample{
    public static void main(String[] args){
        JFrame frame = new JFrame("JDIC API Demo - ChooseBrowserSample");
        frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
        JPanel panel = new JPanel();
        panel.setLayout(new BorderLayout());
        panel.setPreferredSize(new Dimension(700,500));
        BrowserEngineManager bem= BrowserEngineManager.instance();
        bem.setActiveEngine(BrowserEngineManager.MOZILLA);
        IBrowserEngine be=bem.getActiveEngine();
        //the path that constains xpcom.dll
        be.setEnginePath("d:\\mozilla\\bin\\mozilla.exe");
        //For IE, setActiveEngine is enough
        //bem.setActiveEngine(BrowserEngineManager.IE);
        final WebBrowser webBrowser = new WebBrowser();
        try{
            webBrowser.setURL(new URL("http://www.google.com"));
        } catch (MalformedURLException e) {
            e.printStackTrace();
        }
        webBrowser.addWebBrowserListener(new WebBrowserListener(){
            public void downloadStarted(WebBrowserEvent event){;}
            public void downloadCompleted(WebBrowserEvent event){;}
            public void downloadProgress(WebBrowserEvent event){;}
            public void downloadError(WebBrowserEvent event){;}
            public void documentCompleted(WebBrowserEvent event){;}
            public void titleChange(WebBrowserEvent event){;}
            public void statusTextChange(WebBrowserEvent event){;}
        });
        panel.add(webBrowser, BorderLayout.CENTER);
        frame.getContentPane().add(panel,BorderLayout.CENTER);
        frame.pack();
        frame.setVisible(true);
    }
}  

Q. How to get source of the HTML document by browser?
A. Load a page using WebBrowser instance and get the HTML document source by calling the WebBrowser.getContent() method:
public class WebBrowserSample{
    public static void main(String[] args){
        JFrame frame = new JFrame("JDIC API Demo - WebBrowserSample");
        frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);

        panel.setLayout(new BorderLayout());
        panel.setPreferredSize(new Dimension(700,500));
        final WebBrowser webBrowser = new WebBrowser();

            webBrowser.setURL(new URL("http://www.google.com"));
        } catch (MalformedURLException e) {
            e.printStackTrace();
        }
        webBrowser.addWebBrowserListener(new WebBrowserListener(){
            public void downloadStarted(WebBrowserEvent event){;}
            public void downloadCompleted(WebBrowserEvent event){;}
            public void downloadProgress(WebBrowserEvent event){;}
            public void downloadError(WebBrowserEvent event){;}
            public void documentCompleted(WebBrowserEvent event){
                String content = webBrowser.getContent();
                System.out.println("Source of page: ");
                System.out.println(content);
            }

        });
        panel.add(webBrowser, BorderLayout.CENTER);

        frame.setVisible(true);
    }
}  

Q. How to set a series of URL by browser?
A. You can use WebBrowser.synSetURL() method to realize this and it can only be called when initializationCompleted event is fired. This is a new feature and not available from our provided build till now(JDIC_20060308_build). You can get all source java codes from CVS or specific files by http://forums.java.net/jive/thread.jspa?messageID=97561 to get this new API.
public class SeriesBrowserSample{
    public static void main(String[] args){
        JFrame frame = new JFrame("JDIC API Demo - SeriesBrowserSample");
        frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
        JPanel panel = new JPanel();
        panel.setLayout(new BorderLayout());
        panel.setPreferredSize(new Dimension(700,500));
        final WebBrowser webBrowser = new WebBrowser();
        webBrowser.addWebBrowserListener(new WebBrowserListener(){
            public void initializationCompleted(WebBrowserEvent event){;}
                try{
                    webBrowser.syncSetURL(new URL("http://www.baidu.com"));
                    webBrowser.syncSetURL(new URL("http://www.sun.com"));
                    webBrowser.syncSetURL(new URL("http://www.google.com"));
                } catch (MalformedURLException e) {
                    e.printStackTrace();
                } catch (JdicInitException e) {
                    e.printStackTrace();
                }
            }
            public void downloadStarted(WebBrowserEvent event){;}
            public void downloadCompleted(WebBrowserEvent event){;}
            public void downloadProgress(WebBrowserEvent event){;}
            public void downloadError(WebBrowserEvent event){;}
            public void documentCompleted(WebBrowserEvent event){;}
            public void titleChange(WebBrowserEvent event){;}
            public void statusTextChange(WebBrowserEvent event){;}
        });
        panel.add(webBrowser, BorderLayout.CENTER);
        frame.getContentPane().add(panel,BorderLayout.CENTER);
        frame.pack();
        frame.setVisible(true);
    }
}
