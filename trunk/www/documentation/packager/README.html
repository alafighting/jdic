1. Overview

JDIC Packager is a Java-based tool to package JNLP applications into installable packages of MSI/RPM/PKG format , the standard Windows/Linux/Solaris OS package respectively. It's part of JDIC project.

Note: On Windows platform, the generated package will be provided as a bootstrapper executable file.

2. System Requirements

  • Supported platforms
    • Win98/ME/NT/2000/XP/2003, Sun Java Desktop System(JDS) Release 1 or later, SuSE Linux 7.1 or later, RedHat Linux 8 or later, Solaris 8 or later.
  • Additional software
    • Windows Developers need to download and install windows MSI SDK 1.5 to generate MSI packages.
    • End user using Windows98/NT needs MSI service package to install MSI packages. Download the following executable and install.
      Windows NT4 MSI Service Package, Windows 9x MSI Service Package
    • Linux: rpm-build package.
    • J2SE v1.5.0 or later.

3. Binary

JDIC Packager is released both in binary distributions and source distributions. Binary distributions are available both in gzipped tar format (jdic-packager-<version>-bin-<platform>-[arch].tar.gz) and zip format (jdic-packager-<version>-bin-<platform>-[arch].zip) here.

The notation <version> represents the version number, such as 1.0, 1.0.2, and <platform> represents the platform the distribution works on, which is windows, linux or solaris. The [arch] notation is only used for Solaris platform (<platform>=solaris), which is sparc or i586. For the other platforms (Windows/Linux), it's i586 by default.

Extracting a binary distribution will create below directory structure:

jdic-packager-<version>-bin-<platform>-[arch]
+- README.html: this file.
|
+- LICENSE: the license file.
|
+- NOTES: Notes, issues and known bugs.
|
+- bin/: contains Windows specific libraries and running executable.
|  +- jPackager.exe: running executable.
|  +- *.dll (For Windows platform only)
|
+- lib/: contains jar file, Linux/Solaris-specific libraries, and running script.
|  +- jPackager: running script.
|  +- jdic-packager.jar
|  +- *.so (For Linux and Solaris only)
|
+- docs/: contains documentations.
|  +- javadoc/:
|
+- [demo]/: API demo
|
+- [examples]/: contains API example source code and/or binaries.
|
+- [tests]/: contains test case source code and/or binaries.

4. Source Code

JDIC Packager source distributions are available in gzipped tar format (jdic-packager-<version>-src.tar.gz) and zip format (jdic-packager-<version>-src.zip), including the source code for all the supported platforms.

4.1 Prerequisites

The following additional tools and libraries to build the sources (including test code):

  • Jakarta Ant:
    JDIC Packager comes with a simple, Ant-based build system. Obtain and install Ant as outlined in the manual.
  • JUnit:
    JDIC Packager uses JUnit as the test framework and tool. Obtain and install JUnit as outlined in the manual, which is ONLY used to build and run test code.
  • [Windows] Microsoft Visual C++ Compiler :
    The Microsoft Visual C++ 6.0 Professional Edition compiler is required for building the source code. The compiler and tools binaries must be in the PATH.
  • [Windows] Microsoft Windows SDK Update :
    The Microsoft Windows SDK update is required for building the source code, generating the MSI package. The SDK update package should be installed and configured correctly.

4.2. Extract the source code

Extracting a source distribution will create below directory structure:

|  |  +- jPackager.exe: running executable.
|  |  +- classes/, native/
|  |

|     +- jPackager: running script.

4.3. Building

After get and extract a source distribution, below are the basic steps to build it:

1) Get and install the needed software as outlined in Section 4.1, set up them correctly.

2) Change to the top directory of JDIC Packager source code. Run "ant" to see all the available targets. The current available targets are:
   showtargets: the default target, to display all the available targets and their description.
   clean: target to remove all the generated directories and files.
   buildall: target to build all the .jar files, native libraries, javadoc for the current platform.

3) After a successful build, a new dist directory will be created, with all the generated directories and files put there.

4) If any changes are made to the source code, run Ant again. It will perform an incremental rebuild of the target.

5. Usage

Two kinds of usages are provided. Developers could invoke java command to launch the package. Scripts are also provided for direct usage.

5.1 For Java users

  • Add jdic-packager.jar to CLASSPATH.
  • Add <JAVA_HOME>\jre\lib\deploy.jar and <JAVA_HOME>\jre\lib\javaws.jar to CLASSPATH.
  • Add native library (*.dll/*.so) directory to PATH (for Windows) or LD_LIBRARY_PATH (for Unix).
  • Run Java [properties configuration] org.jdesktop.jdic.packager.Jnlp2Msi c:\temp\temp\draw.jnlp

  • Following properties are availabe for configuration

-ResourceDir resourcedir
Specify the directory of the JNLP resource files.
If this option is not set, the parent path (..) of the given JNLP file will be used.
-PackageName packagename
Specify the name of the generated package.
If this option is not set, the given JNLP file name without the suffix .jnlp will be used.
-OutputDir outputdir
Specify the destination path of the generated package.
If this option is not set,
Windows: the current directory (.) will be used.
linux: $HOME/rpm will be used.
Solaris: /var/spool/pkg will be used.
-Version version
Specify the version number of the generated package.
If this option is not set, 1.0 will be used.
-Release release
Specify the release number of the generated package.
If this option is not set, 1 will be used.
-LicenseDir licensedir
Specify the directory of the license files.
If this option is not set, there would be license files used.
 
-BannerJpgFile[Windows Only] banner jpeg file path
Specify the path of the JPEG file to be used as MSI banner picture.
-PanelJpgFile[Windows Only] panel jpeg file path
Specify the path of the JPEG file to be used as MSI panel picture.
-MSSDKPath[Windows Only] Microsoft Windows SDK update installation directory
Specify the Microsoft Windows SDK intallation path. Note: this property must get configured on Windows platform.
-EnableShortcut true/false
Specify whether the generated installation package will create a shortcut during installation. Value: true, false.
-EnableAssociation true/false
???.
-EnableSystemCache true/false
Specify whether whether the installer should get this application installed into Javaws system cache.
5.2 For script users
  • Add jdic-packager.jar to CLASSPATH.
  • Add <JAVA_HOME>\jre\lib\deploy.jar and <JAVA_HOME>\jre\lib\javaws.jar to CLASSPATH.
  • Add native library (*.dll/*.so) directory to PATH (for Windows) or LD_LIBRARY_PATH (for Unix).
  • Run script packager launcher:

    JDIC Packager launcher (jnlp2msi, jnlp2rpm, jnlp2pkg for Windows/Linux/Solaris)

    These launcher program will help to launch JDIC Packager with the required path of the JNLP file and the given options. It's actually a wrapper script for the main class of JDIC Packager. It just pass down the user given path of .jnlp file and relevant options to the main class.

    • Synopsis
        jnlp2msi/jnlp2rpm/jnlp2pkg [options] <JNLP file path>
options
Command-line options in -option value format.
<JNLP file path>
Required path of the JNLP file to be invoked. 
  • Options specification

-rd -resourcedir resourcedir

Specify the directory of the JNLP resource files.
If this option is not set, the parent path (..) of the given JNLP file will be used.

-pn -packagename packagename
Specify the name of the generated package.
If this option is not set, the given JNLP file name without the suffix .jnlp will be used.
 
-od -outputdir outputdir
Specify the destination path of the generated package.
If this option is not set,
Windows: the current directory (.) will be used.
linux: $HOME/rpm will be used.
Solaris: /var/spool/pkg will be used.
 
-v -version version
Specify the version number of the generated package.
If this option is not set, 1.0 will be used.
 
-r -release release
Specify the release number of the generated package.
If this option is not set, 1 will be used.
 
-ld -licensedir licensedir
Specify the directory of the license files.
If this option is not set, there would be license files used.
 
-bjf -bannerjpgfile[Windows Only] bannerjpgfilepath
Specify the path of the JPEG file to be used as MSI banner picture.
 
-pjf -paneljpgjile[Windows Only] paneljpgfilepath
Specify the path of the JPEG file to be used as MSI panel picture.
 
-msd -mssdkdir[Windows Only] Microsoft Windows SDK installation directory
Specify the Microsoft Windows SDK intallation path. Note: this property must get configured on Windows platform.
 
-es -enableshortcut true/false
Specify whether the generated installation package will create a shortcut during installation.
 
-ea -enableassociation true/false
