Example on using JDIC Packager to package a JNLP application

  • Draw - sample JNLP application to package

We take a JNLP application such as Draw as an example to demonstrate how to use the JDIC Packager.  

Draw is a two dimensional drawing program allowing you to create rectangles, polygons, circles, which is created  by Sun. The binary (draw.zip) for this application is available for download here.

We will provide some sample usages to package this Webstart application into native installation packages.

Windows Platform | Linux Platform | Solaris Platform | Verify the Installed Package

  • Steps to package the Draw application into an installable package on Windows Platform

Below are the steps to package Draw into an MSI package (as executable bootstrapper format) on Windows.

1. Download draw.zip from the above link, and unzip it into a temporary directory.

Example:

C:\temp\draw
+- draw.jnlp: 
+- draw.jar: 
+- images/: 
   +- draw.jpg

2. Follow up the JDIC Packager installation guideline in the README file to setup the running environment.

3. Run jnlp2msi.exe.

If we are currently under the directory C:\temp. Running "jnlp2msi.exe" without any given parameter will print help messages.

Run - "jnlp2msi.exe -mssdkdir "C:\Program Files\Microsoft SDK" -resourcedir .\draw -packagename drawpkg -outputdir C:\ -version 2.0 -release 2 .\draw\draw.jnlp".

There should be a new file drawpkg.exe under directory C:\ with version number 2.0 and release number 2.

4. Install the generated package.

Double clicking the generated drawpkg.exe file will install the Webstart application into the system. Snapshot below:

  • Steps to package the Draw application into an installable package on Linux Platform

Below are the steps to package Draw into RPM format package on Linux platform.

1. Download draw.zip from the above link, and unzip it into a temporary directory.

Example:

C:\temp\draw
+- draw.jnlp: 
+- draw.jar: 
+- images/: 
   +- draw.jpg

2. Follow up the JDIC Packager installation guideline in the README file to setup the running environment.

3. Run jnlp2rpm

If we are currently under the directory /temp. Running "jnlp2rpm" without any given parameter will print help messages.

Run - "jnlp2rpm -resourcedir ./draw -packagename drawpkg -outputdir /var -version 2.0 -release 2 ./draw/draw.jnlp".

There should be a new file drawpkg-2.0-2.rpm under directory /var.

4. Install the generated package

Change to root, run rpm installation command to install the generated package. For example

#rpm -i draw-2.0-2.rpm

  • Steps to package the Draw application into an installable package on Solaris Platform

Example:

C:\temp\draw
+- draw.jnlp: 
+- draw.jar: 
+- images/: 
   +- draw.jpg

2. Follow up the JDIC Packager installation guideline in the README file to setup the running environment.

3. Run jnlp2pkg

If we are currently under the directory /temp. Running "jnlp2pkg" without any given parameter will print help messages.

Run - "jnlp2pkg -resourcedir ./draw -packagename drawpkg -outputdir /var -version 2.0 -release 2 ./draw/draw.jnlp".

There should be a new directory drawpkg under directory /var with version number 2.0 and release number 2..

4. Install the generated package

Change to root, run pkgadd installation command to install the generated package. For example

#pkgadd -d /var draw
