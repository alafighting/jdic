Setup An Incubator Project

Below are detailed instructions for the incubator project proposer(s) and JDIC owners to setup a new incubator project in the Incubator Projects section of JDIC. This doc consists of two sections: the For incubator project proposers section, and the For JDIC Owners section, which outline the tasks for the project proposers and JDIC project owners respectively.

For incubator project proposers:

Note: If you are proposing one single method, one single class, type of APIs, please apply for a "javadesktop>swinglabs>jdic.Misc Incubator Project Developer" role for the Misc incubator project and add your APIs to that incubator project. Or else, you need to apply for a separate incubator project following below instructions:

  • Post your proposal to the JDIC Forum.
    The post should be entitled like "[Incubator Project Proposal] XXX". It should give a brief introduction of the problem and a draft specification. This helps to get the feedback from the community, and align efforts with other individual developers and 3rd party projects in the early beginning.
  • Get the JDIC owners' approval. 
    They'll then set up the project following below For JDIC owners section.
  • Finalize the package name and API specification.
    The specs need to be discussed and agreed with among the developers and JDIC owners. The API specification could contain simply one API, or a couple of APIs. Refer to the Java package naming convention while specifying the package name.
  • Check in the code and docs to the CVS repository.
    At least, there needs an index.html (README) file, and a javadoc link, before or after the code is ready. Please add/import the javadoc folder as [jdic]/www/incubator/<modulename>/javadoc, then the javadoc link should be like: https://jdic.dev.java.net/nonav/incubator/<modulename>/javadoc/. DO NOT miss the "nonav" part in the URL link.

    Follow the same file structure of JDIC and all the incubator project source, which makes it easy to be integrated into the central JDIC source someday. A suggestion is to download a JDIC or incubator project source release, and change the source.
  • Release a source or binary download.
    After the owners setup the incubator project, there is a folder for the incubator project in the Document & files page to hold downloads. There is no way to grant the upload permission to the incubator project owners, unlike the source or www modules in the CVS repository. If you release/remove a download by adding/deleting new files or folders, there'll be a "suggest to upload/delete file/folder" notice sent to the JDIC owners, which will be approved shortly.

For JDIC owners:

Once the JDIC owners approve the incubator project proposal, follow below instructions to setup the project. To make it clear, we take the process of setting up the SystemInfo incubator project as an example.

  • Make sure the contributors/owners of the incubator project sign the JCA form according to the instructions.
  • Make sure the incubator project holds or refers to exactly the same license terms as the JDIC project.
  • Add the source code and WWW documents of the incubator project to the JDIC CVS repository.
    • Create a CVS module for the source code of the incubator project in the CVS repository: [jdic]/src/incubator/<modulename>. 
      For the SystemInfo incubator project, it's: [jdic]/src/incubator/systeminfo. 
    • Add/Import the local source code folder of the incubator project into the created CVS module for source code. Use command: "cvs import jdic/src/incubator/<modulename> <Vendor tag> <Release tag> (in directory C:\Temp\<modulename>)".
       For SystemInfo, it's "cvs import jdic/src/incubator/systeminfo <Vendor tag> <Release tag> (in directory C:\Temp\systeminfo)".
    • Create a CVS module for the WWW documents of the incubator project in the CVS repository: [jdic]/www/incubator/<modulename>.
      Add or import WWW documents, including one index.html file and javadoc files, to that module.
  • Grant the access/write permissions to the incubator project owners.
    • Add new resource patterns for the incubator project: 
      JDIC homepage -> Editor project -> Edit project-specific resource patterns -> Add new resource pattern.
    • Add a new project role for the incubator project owner(s) with necessary permissions: 
      JDIC homepage -> Editor project -> Edit project-specific roles -> Add new project role.
    • Grant the created project role to the incubator project's owner(s).
  • Create a subfolder under the "incubator" folder at the Documents & files link.
     For the SystemInfo incubator project, it's: "systeminfo".
  • Add the incubator project to the Incubator Projects section in the JDIC homepage.
  • Announce the incubator project.
    • Give an announcement to the JDIC Forum.
    • Give an announcement to the mailing list announce@jdic.dev.java.net.
    • Give announcements to somewhere else if necessary.
