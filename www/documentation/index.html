The documentation for the JDIC project falls into two categories:

  • JDIC API
  • JDIC Packager

JDIC API

The JDIC API provides an interface from applications to the native desktop.

  • API Documentation (javadoc): Browsable API specification. You can also download it for offline viewing. The JDIC API contains three packages:
    • org.jdesktop.jdic.browser (browser component): Provides an AWT component to enable the embedding of the desktop's web browsing component into Java applications and applets. The API supports URL navigation, a basic history mechanism, and progress notification.  
    • org.jdesktop.jdic.desktop: An API to automate [PENDING: words seem to be missing here: "communicating with"? "using services of"?] native applications, including the document viewer applications associated with a particular file type, the system browser, and the system mailer.
    • org.jdesktop.jdic.filetypes: An API to associate document viewer applications with file types.
  • Other Specifications: Detailed requirement and implementation specifications.
  • README.html: Complete installation and configuration notes for the JDIC API. [PENDING: included in the JDIC downloadable package?]
  • API Examples: Examples of how to use the JDIC API.
  • Deployment: Tips for using Java Web Start to deploy an application that uses the JDIC API.

JDIC Packager

JDIC Packager is a tool for putting JNLP applications into installable packages of the standard Windows, Linux, and Solaris formats — MSI, RPM, and PKG, respectively.
  • README.html: Complete installation and configuration notes for JDIC Packager.
  • Example: Example and detailed steps using JDIC Packager to package a JNLP application.
