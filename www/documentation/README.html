1. Overview

JDIC API is part of JDIC project and is composed of below Java packages:

org.jdesktop.jdic.desktop: APIs to automate native applications, including the document viewer applications associated with a particular file type, the system browser and the system mailer.

org.jdesktop.jdic.filetypes: APIs to associate document viewer applications with filetypes.

org.jdesktop.jdic.browser: Provides an AWT component to enable the embedding of the desktop's web browsing component into Java applications and applets. The APIs support URL navigation, a basic history mechanism and progress notification. 

2. System Requirements

  • Supported platforms
    • Win98/ME/NT/2000/XP/2003, Linux SuSE 7.1 or later, RedHat 8 or later, Solaris 8 or later.
  • Additional software
    • Windows: Internet Explorer 5 or later, or Mozilla 1.4 or later.
    • Linux/Solaris: Gnome 2.0, Ximian Evolution 1.4 or later, Mozilla 1.4 or later.
    • J2SE v1.4.2 or later.

3. Binary

JDIC is released in both binary distributions and source distributions. Binary distributions are available in both gzipped tar format (jdic-<release number>-bin-<arch>.tar.gz) and zip format (jdic-<release number>-bin-<arch>.zip) here.

Extracting a binary distribution will create below directory structure:

+- README.html: this file.
|
+- LICENSE: the license file.
|
+- NOTES: Notes, issues and known bugs.
|
+- bin/: Windows specific libraries.
|  +- *.dll (For Windows platform only)
|
+- lib/: jar file and Linux/Solaris-specific libraries.
|  +- jdic.jar, *.so (For Linux and Solaris only)
|
+- docs/: documentations.
|  +- javadoc/:
|
+- demo/: API demo
|
+- [tests]/: test case source code and/or binaries.

4. Source Code

JDIC source code could be checked out from CVS tree. 

And JDIC source distributions are available both in gzipped tar format (jdic-<release number>-src.tar.gz) and zip format (jdic-<release number>-src.zip) here.

4.1. Prerequisites

The following additional tools and libraries are needed to build the JDIC source code:

  • Jakarta Ant:
    JDIC comes with a simple, Ant-based build system. Obtain and install Ant 1.5.4 or later as outlined in the manual.
  • JUnit:
    JDIC uses JUnit as the test framework and tool. Obtain and install JUnit as outlined in the manual, which is ONLY used to build and run test code.
  • Windows:
    • Microsoft Visual C++ Compiler
      The Microsoft Visual C++ 6.0 Professional Edition compiler is required.
    • Microsoft Platform Software Development Kit (SDK)
      It can be downloaded from here.
  • Linux:
    • GNU Make
      GNU make version 3.78.1 or later is required, which can be downloaded from GNU make web site.
    • GCC Compiler
      GNU gcc version 3.2.1-7 or later is required, which can be downloaded from GCC home page
  • Solaris:
    • GNU Make
      GNU make version 3.78.1 or later is required, which can be downloaded from GNU make web site.
    • Sun ONE Studio Compilers (formerly Forte Tools)
      Forte 6 Update 2 or later Sun ONE Studio 8, Compiler Collection, which could be found here or from Sun ONE Studio Developer Tools web site.

4.3. Building

After get and extract a source distribution, below are the basic steps to build it:

  • Get and install the needed software as outlined in Section 4.1, set up them correctly.
  • Get and build Mozilla 1.4 or later source code. Please refer to http://www.mozilla.org/, and http://www.mozilla.org/build/ for where to download and how to build it. And refer to a FAQ in case of problems. 
    Setup the environment variable MOZILLA_SRC_HOME to the top directory of Mozilla source code.
  • Change to the top directory of JDIC source code. Run "ant" to see all the available targets. The main available targets are:
       showtargets: the default target, to display all the available targets and their description.
       clean: target to remove all the generated directories and files.
       buildall: target to build all the .jar files, native libraries, javadoc for the current platform.

After a successful build, a new dist directory will be created, with all the generated directories and files. If any changes are made to the source code, run Ant again. It will perform an incremental rebuild of the target.

5. Usage

  • Add jdic.jar to CLASSPATH.
  • Add the directory of native libraries (*.dll or *.so) to PATH (on Windows) or LD_LIBRARY_PATH (on Linux/Solaris).
  • Specific usage settings to use package org.jdesktop.jdic.browser
    • Windows
      - Add <JAVA_HOME>\jre\bin to PATH.
      - Add the directory of native executable files (*.exe) to PATH.
      - By default, the system default browser (Mozilla or Internet Explorer) will be embeded. If Mozilla is the embeded(default) browser:
        -- Set the environment variable MOZILLA_FIVE_HOME to specify the path where Mozilla 1.4 (or later) executables locates.
        -- Copy MozEmbed.exe file to MOZILLA_FIVE_HOME directory.
        -- Add MOZILLA_FIVE_HOME to PATH.
    • Linux/Solaris
      -       Add the directory of native exectuable files (MozEmbed-linux-gtk*) to PATH.
      - Set the environment variable MOZILLA_FIVE_HOME to point to the absolute path of the Mozilla 1.4 (or later) executable file.
        Notice: on Sun JDS, this is /usr/lib/mozilla-1.4.
      - Add MOZILLA_FIVE_HOME to LD_LIBRARY_PATH.
