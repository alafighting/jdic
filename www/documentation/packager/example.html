Example of using JDIC Packager to package a JNLP application

Draw - sample JNLP application

We'll use the JNLP application Draw to demonstrate how to use the JDIC Packager.  

Draw is a Sun-provided two dimensional drawing program allowing you to create rectangles, polygons, and circles. The binary is here: draw.zip.

This page shows how to package JNLP applications such as Draw into native installation packages.

Windows | Linux | Solaris | Verify the Installed Package

Steps to package the Draw application into an installable package on Windows

The following steps package Draw into an MSI package (as executable bootstrapper format) on Windows.

1. Download draw.zip and unzip it into a temporary directory.

Example:

C:\temp\draw
+- draw.jnlp: 
+- draw.jar: 
+- images\: 
   +- draw.jpg

2. Follow the JDIC Packager installation guidelines in the README file to set up the running environment.

3. Run jnlp2msi.exe.

Assuming the current directory is C:\temp, run the following command:

jnlp2msi.exe -mssdkdir "C:\Program Files\Microsoft SDK" -resourcedir .\draw -packagename drawpkg -outputdir C:\ -version 2.0 -release 2 .\draw\draw.jnlp

The result is a new file drawpkg.exe under directory C:\ with version number 2.0 and release number 2.

Note: Running jnlp2msi.exe without any parameters prints help messages.

4. Install the generated package.

Steps to package the Draw application into an installable package on Linux

The following steps package Draw into an RPM format package on the Linux platform.

1. Download draw.zip and unzip it into a temporary directory.

Example:

/tmp/draw
+- draw.jnlp: 
+- draw.jar: 
+- images/: 
   +- draw.jpg

2. Follow the JDIC Packager installation guidelines in the README file to set up the running environment.

3. Run jnlp2rpm.

Assuming the current directory is /tmp, run the following command:

jnlp2rpm -resourcedir ./draw -packagename drawpkg -outputdir /var -version 2.0 -release 2 ./draw/draw.jnlp

The result is a a new file drawpkg-2.0-2.rpm under the /var directory.

Note: Running jnlp2rpm without any parameters prints help messages.

4. Install the generated package.

Change to root, and run the rpm installation command to install the generated package. For example

#rpm -i draw-2.0-2.rpm

Steps to package the Draw application into an installable package on Solaris

The following steps package Draw into a PKG format package on the Solaris platform.

1. Download draw.zip and unzip it into a temporary directory.

Example:

/tmp/draw
+- draw.jnlp: 
+- draw.jar: 
+- images/: 
   +- draw.jpg

2. Follow the JDIC Packager installation guidelines in the README file to set up the running environment.

3. Run jnlp2pkg.

Assuming the current directory is /tmp, run the following command:

jnlp2pkg -resourcedir ./draw -packagename drawpkg -outputdir /var -version 2.0 -release 2 ./draw/draw.jnlp

The result is a new directory drawpkg under the /var directory with version number 2.0 and release number 2.

Note: Running jnlp2pkg without any parameters prints help messages.

4. Install the generated package.

Change to root, and run the pkgadd installation command to install the generated package. For example

#pkgadd -d /var draw
