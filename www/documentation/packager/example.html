Example of using JDIC Packager to package a JNLP application

Below give the implementation details of using JDIC Packager to package a sample JNLP application.

  • Draw - the sample JNLP application to package

We take a JNLP application Draw as an example to demonstrate the usage of JDIC Packager.  

Draw is a two dimensional drawing program allowing you to create rectangles, polygons, circles, which is created  by Sun. The binary (draw.zip) for this application is available for download here.

  • Steps of packaging the Draw application into an installable package

Below are the steps to package Draw into an MSI package on Windows. To package it into a RPM package on Linux or PKG package on Solaris follows the same steps.

1. Download draw.zip from the above link, and unzip it into a temporary directory, for example, C:\temp\draw. 

There are below JNLP files under this directory: 

C:\temp\draw
+- draw.jnlp: 
+- draw.jar: 
+- images/: 
   +- draw.jpg

2. Follow up the JDIC Packager usage guideline to setup the running environment for jPackager.exe.

3. Run jPackager.exe.

Suppose we are currently under directory C:\temp. We give below values for the required parameter and options.

  • <JNLP file path> = .\draw\draw.jnlp: this is the path of the JNLP file.
  • -resourcepath .\draw : since all the jnlp resource files (.jar file and image files) are located under this path.
  • -packagename drawpkg : if this option is not used, draw will be used by default. 
  • -packagepath C:\ : if this option is not used, the current directory (C:\temp) will be used.
  • -version 2.0 :  if this option is not used, 1.0 will be used.
  • -release 2 : if this option is not used, 1 will be used.
  • -licensepath : we do not use this option for this example.

So, the complete command line is:

jPackager.exe -resourcepath .\draw -packagename drawpkg -packagepath C:\ -version 2.0 -release 2 .\draw\draw.jnlp

Hopefully, we'll get an installable drawpkg.msi file under directory C:\.
