General

Q. What is SaverBeans?
A. SaverBeans is a collection of platform independent screensavers and a set of tools to help developers build them.

Q. What is the SaverBeans SDK?
A. It is a software development kit for developing platform-independent screensavers, and it is part of the JDIC project.

Q. What is JDIC?
A. JDIC is short for JDesktop Integration Components.  It "aims to make Java technology-based applications ("Java applications") first-class citizens of current desktop platforms without sacrificing platform independence."

Q. What is the SaverBeans Screensaver Pack?
A. It is a collection of community-contributed screensavers built with the SaverBeans SDK.

Q. Where can I download the SaverBeans SDK and SaverBeans Screensaver Pack?
A. Click to download the SaverBeans SDK and SaverBeans Screensaver Pack.

Q. How stable is the SaverBeans SDK?
A. The SaverBeans SDK is currently an early access release and is likely to have a number of bugs.  We've tested it on various flavors of Windows and Unix and have had success, but true stability won't come until more people use it and report bugs.

Q. I've written a screensaver.  Where can I submit it?
A. Go to the screensavers project and follow the instructions there.

Q. I've found a bug in a screensaver or the SaverBeans SDK.  Where do I report it?
A. If the bug is in a particular screensaver, report it here.  If you are sure the bug is in the SDK itself (e.g. incorrect code generated for a particular platform), report it here.  If you're not sure, use the first link.

Q. What forums are available?
A. There is a general screensavers discussion forum for screensavers developed using the SDK.

Installing Screensavers

Q. How do I install a SaverBeans screensaver on Windows?
A. First, download the screensaver.  For Windows, you should only download the files that end in -win32.zip.  Then, copy all the files in the .zip (except README.txt) to your Windows system directory (usually C:\WINDOWS\SYSTEM32).  That should do the trick.  See the README.txt file that came with the screensaver for more details.

Q. How do I install a SaverBeans screensaver on Unix (Linux, Solaris, JDS, etc.)?
A. First, download the screensaver.  For Unix, you should only download the files that end in -unix.zip.  Then, follow the instructions in the README.txt.  You will have to copy files to various places in your system to install the screensaver.

Running Screensavers

Q. I tried to run the 'blackglass' screensaver but the screen looks like it's chopped in half.  What happened?
A. This is how it's supposed to look - let it run for a little while and the effect will become clear.  The beans at the top of the screen are going in a circle that's bigger than the screen and the beans on the bottom of the screen are simply a reflection.

Q. Is it possible to change which Java Virtual Machine (JVM) the screensaver will use?
A. On Unix, the virtual machine is automatically detected using the steps outlined in the README.  If this automatic detection fails, you can override it by passing -jdkhome to the screensaver.  On Windows, the screensaver will use the "HKEY_LOCAL_MACHINE\Software\JavaSoft\Java Runtime Environment\JavaHome" registry setting to determine which JVM to use.  This is automatically set by the JVM installer.

Q. My screensaver is running a bit choppy on Unix, especially in the ScreensaverFrame in debug mode.
A. Make sure you are running J2SE 1.5.0 or greater, even when you are debugging.  Earlier versions of the JDK may have choppy timing for animation on Linux.

Q. Screensavers are crashing on Windows. I'm getting something like "The instruction xxxxxxxx referenced memory at xxxxxxx, the memory count not be read, click Ok to terminate", or something similar. What can I do to figure out what's wrong?
A. This can be caused by a number of different problems. To find out why this is happening, you need to run a special debug screensaver that writes a debug log in the C:\WINDOWS\TEMP directory. Follow these instructions and post the results to the screensavers.dev.java.net forum for help.
  1. Download the special debug screensaver.
  2. Unzip to a new folder on your hard drive. Make sure it's a new folder and you do not overwrite any existing files.
  3. Right-click on debugsaver.scr (the one with the bean icon) and click "Test".
  4. Let the screensaver run for a bit. You should see a blue line with the text "DEBUG SCREENSAVER" at one end.
  5. Exit the screensaver by moving the mouse or pressing a key.
  6. Check for a file called C:\WINDOWS\TEMP\saverbeans-log.txt and look for any error messages. Post the log to the forum if you have any questions.

Writing Screensavers

Q. How do I write a screensaver configuration XML file?  Is there a DTD or schema available?
A. The screensaver configuration XML file syntax is the same as the xscreensaver configuration file syntax. I've obtained permission from the author to post it locally, but it can also be found in the hacks/config/README file in the xscreensaver distribution. There are also various examples in that directory.  In order for the SDK to work correctly, you must always have the -root, -jar, -class and -jdkhome options, and the -jar and -class must be hard-coded via a <command> tag to the appropriate values.  See the bouncingline sample for details.

Q. My screensaver is throwing an exception - how do I access the stack trace?
A. The SaverBeans SDK uses the J2SE logging API.  In most cases (e.g. if the exception is thrown during the paint() method) you should be able to retrieve the stack trace by adjusting your JVM logging configuration as follows:
  1. Edit your $JAVA_HOME/jre/lib/logging.properties file
  2. Change the line:
        handlers= java.util.logging.ConsoleHandler
    to:
        handlers= java.util.logging.FileHandler, java.util.logging.ConsoleHandler
  3. At the end of the file, add the following line:
        org.jdesktop.jdic.screensaver.level=ALL
  4. Restart the screensaver and look for a file called java0.log in your home directory.  That file should contain your stack trace and other debugging information.
Q. How do I log information from my screensaver?
A. The SaverBeans SDK uses J2SE logging.  Follow the above instructions to enable logging to a file.  Then, use code similar to the following to log information from your screensaver:
import java.util.logging.*;
...
Logger logger = Logger.getLogger("org.jdesktop.jdic.screensaver");
logger.log(Level.INFO, "Log message.");
Q. How can I debug my screensaver using my IDE?
A. With SaverBeans SDK 0.1.1 or greater there is a new ScreensaverFrame class that lets you run your screensaver in Java instead of being launched in native mode.  This makes it easy to debug.  In fact, the startup kit now includes a 'debug' ant task by default, which should make debugging much easier (just connect via JPDA from your favorite IDE, such as NetBeans).
