Java Desktop Integration Components

Java Desktop Integration Components Version (JDIC) 0.9. This project aims to make Java applications first citizens of current desktop platforms without sacrificing platform independence. It provides Java applications with access to facilities provided by the native desktop such as the desktop mailer, the html rendering engine or registered document viewing applications. Additionally it provides the mechanisms by which Java applications can integrate into the native Desktop such as registering Java applications as document viewers on the desktop and creating desktop installer packages containing Java applications. JDIC is composed of four sub-projects: Desktop, Filetypes, Packager and Browser:

  • Desktop provides APIs to automate native applications, including the document viewer applications associated with a particular file type, the system browser and the system mailer.

  • Filetypes provides APIs to associate document viewer applications with filetypes

  • Packager provides tools to package JNLP applications into installable packages of MSI/RPM/PKG format , standard Windows, Linux and Solaris Operating System packags respectively.

  • Browser provides APIs to enable the embedding of the desktop's web browsing component into Java applications. The APIs support URL navigation, a basic history mechanism and progress notification.
