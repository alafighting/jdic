Setup An Incubator Project

Below are detailed instructions for the incubator project proposer(s) and JDIC owners to setup a new incubator project in the Incubator Projects section. This document consists of two sections: the For incubator project proposers section, and the For JDIC Owners section, which outline the tasks for the project proposers and JDIC project owners respectively.

For incubator project proposers:

Note: If you are proposing one single method or one single class type of APIs, please apply for a "javadesktop>swinglabs>jdic.Misc Incubator Project Developer" role for the Misc incubator project and add your APIs to that incubator project. Or else, please apply for a separate incubator project following below instructions:

  • Post your proposal to the JDIC Forum.
    The post should be entitled like "[Incubator Project Proposal] XXX", which a brief introduction of the idea and the problem. A draft API specification would be helpful. At the same time, enter an issue in Issue Tracker to keep track of it, which can be used to share prototype code and patches. 

    This helps to get the feedback from the community, and align efforts with other individual developers and 3rd party projects in the early beginning.
  • Get the JDIC owners' approval. 
    They'll then set up the project following below For JDIC owners section.
  • Finalize the project name, package name and API specification.
    Refer to the Java package naming convention while specifying the package name. The specs need to be discussed and agreed with among the developers and JDIC owners. 
  • Check in the code and docs to the CVS repository.
    For each project, there is an index.html (README) file, which should provide a general description of the project, a javadoc link, souce code building instructions and relevant information. Add/import the javadoc folder as [jdic]/www/incubator/<modulename>/javadoc, then the javadoc link should be like: https://jdic.dev.java.net/nonav/incubator/<modulename>/javadoc/. DO NOT miss the "nonav" part in the URL link.

    Follow the same file structure of JDIC and all the incubator projects, which makes things easy while integrating them together. A suggestion is to download a JDIC or incubator project source release, and change the source.

    The code should follow the Code Conventions for the Java Programming Language by Sun. To make things easier, some source code formatter tools like Jalopy or Checkstyle can be used.
  • Release a source and binary download.
    After the incubator project is setup, there is a folder for the incubator project in the Document & files page to hold downloads. There is no way to grant the upload permission to the incubator project owners. If you release/remove a download by adding/deleting new files or folders, there'll be a "suggest to upload/delete file/folder" notice sent to the JDIC owners, which will be approved shortly.

The source and binary releases (in .zip or other format) should follow the same file structure as the JDIC releases. The source or binary files don't need to copied manually into one directory, instead, it can be done with targets in the build.xml file.

  • Make a Java Web Start demo
    A Java Web Start demo helps users to try out the API much easier. For general information on Java Web Start, refer to Java Web Start website. You can also read JDIC Deployment page to make a Java Web Start demo. The demo will be added to the Demos section of JDIC homepage.

For JDIC owners:

JDIC owners follow below instructions to setup a new incubator project. To make it clear, we take the process of setting up the SystemInfo incubator project as an example.

  • There is a signed JCA form from the owners of the incubator project according to the instructions.
  • The incubator project holds the same JDIC license terms.
  • Add the source code and WWW documents to the CVS repository.
    • Create a CVS module for the source code of the incubator project in the CVS repository: [jdic]/src/incubator/<modulename>. 
      For the SystemInfo incubator project, it's: [jdic]/src/incubator/systeminfo. 
    • Add/Import the local source code folder of the incubator project into the created CVS module for source code. Use command: "cvs import jdic/src/incubator/<modulename> <Vendor tag> <Release tag> (in directory C:\Temp\<modulename>)".
       For SystemInfo, it's "cvs import jdic/src/incubator/systeminfo <Vendor tag> <Release tag> (in directory C:\Temp\systeminfo)".
    • Create a CVS module for the WWW documents of the incubator project in the CVS repository: [jdic]/www/incubator/<modulename>.
      Add or import WWW documents, including one index.html file and javadoc files, to that module.
  • Grant the access/write permissions to the incubator project owners.
    • Add new resource patterns for the incubator project: 
      JDIC homepage -> Editor project -> Edit project-specific resource patterns -> Add new resource pattern.
    • Add a new project role for the incubator project owner(s) with necessary permissions: 
      JDIC homepage -> Editor project -> Edit project-specific roles -> Add new project role.
    • Grant the created project role to the incubator project's owner(s).
  • Create a subfolder under the "incubator" folder at the Documents & files link.
     For the SystemInfo incubator project, it's: "systeminfo".
  • Add the incubator project to the Incubator Projects section in the JDIC homepage.
  • Announce the incubator project.
    • Give an announcement to the JDIC Forum.
    • Give an announcement to the mailing list announce@jdic.dev.java.net.
    • Give announcements to somewhere else if necessary.
